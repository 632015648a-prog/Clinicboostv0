# Pilot Definition — ClinicBoost

## Qué entendemos por “piloto asistido”
Un piloto asistido no significa producto perfecto.
Significa que una clínica real puede usar ClinicBoost en un caso controlado, con supervisión del fundador/equipo, sin romper la operativa básica.

## Qué debe estar listo para considerar el piloto viable
### Núcleo obligatorio
- login funcional
- dashboard usable
- inbox usable
- Flow 00 funcional
- Flow 01 funcional
- mensajes visibles y trazables
- operativa humana mínima
- aislamiento tenant correcto
- variables locales/staging claras

### Operativa humana mínima
- ver conversaciones
- abrir detalle
- marcar estados
- añadir nota útil
- enviar mensaje manual
- detectar conversaciones que necesitan intervención

### Confiabilidad mínima
- health checks básicos
- logs suficientes
- errores visibles
- entorno local estable
- staging interno entendible

## Qué NO exige el piloto
- Flows 02-07 completos
- automatización total
- observabilidad enterprise completa
- producto listo para escalar a muchos tenants
- UX totalmente refinada
- sistema comercial totalmente cerrado

## Criterio de éxito del piloto
El piloto es válido si la clínica puede usar el caso principal de forma controlada y entender el valor del producto sin que el equipo esté apagando fuegos constantemente.

## Regla práctica
Si algo no ayuda a que una clínica use el caso principal con confianza, no es prioridad inmediata.
